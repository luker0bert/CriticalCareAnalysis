Predicting Length of Stay

Using the MIMIC Critical Care Database

Overview & Hypothesis

The MIMIC-III (Medical Information Mart for Intensive Care) Clinical Database contains comprehensive clinical data from 40,000+ Intensive Care Unit (ICU) patients. This data is analyzed for predictive analysis and machine learning purposes.

  • Can this model improve patient care and potentially save lives?
  • Can we prioritize important clinical measures at the time of patient admission?
  • Can we understand and optimize patient throughput?

MIMIC Database: Table Structure

  • Ho: The first blood lab events are not good predictors for classifying likelihood of mortality and subsequent information.
  • Ha: We can classify patients as likely to die and predict their length of stay (LOS) or days to death (DTD) based on their first blood labs on admission.

Strategy

We are trying to classify mortality probability given a set of features From there, we are trying to predict LOS and TTD depending on the classification with the same set of features Therefore the strategy is: use a neural network and a multi-variate regression via a Supervised Learning (ML) model to classify and predict

The MIMIC Project

Collaborative research - MIMIC is an openly available dataset developed by the MIT Lab for Computational Physiology, comprising deidentified health data associated with ~40,000 critical care patients. It includes demographics, vital signs, laboratory tests, medications, and more. Citation - MIMIC-III, a freely accessible critical care database.
