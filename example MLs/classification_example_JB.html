Methodology¶

I follow an analytics paradigm when solving any problem end-to-end answering why?, how?, and what?. Doing this ensures three things: 1) process efficiency 2) error-reduction in analysis 3) and most importantly, a value-driven outcome.

Why¶

Decompose the Ask, and understand the data
The ask was given. Build two different Machine Learning (ML) models and create predictions based on test data.

Identify the data source
The training and test sets were provided in two separate CSV files.

Strategy
Because we know the data labels and the requested output, this is a Supervised ML problem: We are trying to predict Y given 100 features, X0 - x99.

How¶

Data retrieval plan and retrieval
The data was provided. Pandas will be used to read the CSV in Python and gain insights about the data. However, some data transformation is required to get the data in a format needed for our ML model. That is the extent of ETL for this project.

Assemble and clean
There are several entries where values have been deleted to simulate dirty data. Please clean the data with whatever method(s) you believe is best/most suitable. Note that some of the missing values are truly blank (unknown answers). The simplest way to accomplish this is to dropna for any record containing a null. In doing so, we still have a large sample of the set. Afterwards, dummying the qualitative data is required: out of the 100 features, 39 additional features were created

What¶

Analysis

  • Process: Load->Prepare->Model->Train->Predict->Visualize
  • Because the instructions specify only 2 models, I chose these due to their simplicity and ability to perform the task required: 1) LogisticRegression 2) RandomForestRegressor.
  • Each model produces a csv file of probability predictions.

Limitations & future considerations

  • Experiment with using an imputer instead of simply dropping nulls.
  • Note that because exercise_03_test.csv removed y (aka y_test), it is impossible to calculate model accuracy or residuals.
  • This code takes approximately 6 minutes to run top to bottom. This is a potential concern for scaling.
  • Because the instructions specify to use only 2 models, I chose logistic regression and random forest classifier due to their simplicity and ability to perform the task required. Looking at other models and doing a series of nearest neighbor classifications would be useful as well.

Results¶

Both models produce csv files of the probabilities utilizing the built-in predict_proba(X) method. The outputs differ between models, but by visualizing, we can see a relationship between the two prediction sets.
